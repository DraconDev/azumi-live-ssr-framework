# Azumi — Master TODO

> Full-stack Rust web framework. Simple, secure, compiled.
> Born from: *"My backend is Rust — why is my frontend the thing that breaks?"*

---

## The North Star

A Rust full-stack framework where:
- **No second language** for the frontend. One compiler from DB schema to DOM attribute.
- **No WASM download tax.** ~3KB runtime. HTML is truth. Interactivity is surgical.
- **Compiler catches frontend bugs** — CSS class typos, bad nesting, wrong attributes blocked at compile time.
- **Feels simple.** Not Leptos-complex. Not Dioxus-heavy. Just Rust + `html!` + attributes.

**Competitor positioning:**
| Competitor | Why not |
|-----------|---------|
| Next.js / SvelteKit | Language boundary at fetch(). Two build systems, two ecosystems. |
| Leptos / Dioxus | WASM download tax (~150KB). Complex reactivity for simple forms. |
| HTMX | No type safety — everything is strings. Compiler can't help. |
| Maud | No interactivity story. Just templates. |

**Differentiator:** Compile-time CSS-HTML co-validation + zero-hydration + signed state + ~3KB runtime.

---

## Non-Goals (Things We Will NOT Do)

| Temptation | Why Not |
|-----------|---------|
| ORM integration | Users pick sqlx/diesel/sea-orm independently |
| Authentication package | Too many variants. Signed state is the foundation. |
| CSS framework integration | Validation is the right abstraction level |
| WASM/SSR hybrid | Defeats "no WASM" philosophy. Point users to Leptos. |
| Client-side SPA routing | MPA + Idiomorph is simpler and faster |
| File-system routing | `routes!` macro is simpler, type-checked, no build scripts |
| Custom build tooling | `cargo` works fine |
| Separate template language | `html!` IS the template — it's Rust |

## Phase 1: API Polish — Make It Feel Simple (Next 2-3 Sessions)

### P1.1 — Merge `#[live]` + `#[live_impl]` into One Attribute

**Current (two macros, confusing):**
```rust
#[azumi::live] pub struct Counter { pub count: i32 }
#[azumi::live_impl(component = "counter_view")]
impl Counter { pub fn increment(&mut self) { self.count += 1; } }
```

**Target (one macro):**
```rust
#[azumi::live(component = "counter_view")]
pub struct Counter {
    pub count: i32,
    pub fn increment(&mut self) { self.count += 1; }
}
```

Keep `#[live_impl]` working for backward compat, mark `#[doc(hidden)]`.
**Effort:** 1-2 sessions | **Deps:** None (but best before P0.3)

---

### P1.2 — Reduce Visible API Surface

**Target: 10 macros → 5 visible, 6 traits → 2 visible, 33 fns → ~15 visible**

**Keep public:** `html!`, `#[component]`, `#[live]` (merged), `#[action]`, `json_data!`
**Hide `#[doc(hidden)]`:** `head!`, `#[page]`, `#[live_impl]`, `#[predict]`, `from_fn`, `from_fn_once`, `session_cleanup_script`, `Escaped`, `RenderWrapper`, `FnComponent`, `FnOnceComponent`, `HotReloadClosure`, `LiveStateMetadata`, `FallbackRender`
**Prelude shrinks:** Remove `head`, `live_impl`, `page`, `predict`, `session_cleanup_script`, `from_fn`, `FnComponent`

Full cheat sheet in the extended version of this document.
**Effort:** 1 session | **Deps:** P1.1

### P1.3 — Versioning Promise

**Problem:** v47 in 6 months = breaking every ~4 days = zero trust.

**What:** Ship v48.0.0 with semver promise:
- Major = actual breaking changes (API removal, behavior change)
- Minor = new features, backward compatible
- Patch = bug fixes only
Add to CHANGELOG.md and README.
**Effort:** 1 changelog entry | **Deps:** None
