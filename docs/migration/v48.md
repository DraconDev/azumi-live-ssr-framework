# Azumi v48.0.0 Migration Guide

> Breaking changes from v47.x. Read before upgrading.

---

## Breaking: `class={"literal"}` Now a Compile Error

`class={"my-class"}` with a string literal in braces is now banned, same as `class="my-class"`. Both bypass compile-time class validation.

**Before (v47):**
```rust
html! {
    <div class={"button"}>"Click"</div>  // compiled in v47, no validation
}
```

**After (v48):**
```rust
// ✅ Option 1: Use a variable (validated against <style> blocks)
html! {
    <div class={btn_class}>"Click"</div>
}

// ✅ Option 2: Use class:external for third-party component CSS
html! {
    <div class:external="payment-widget">"Click"</div>
}
```

**Migration:** Search your codebase for `class={"...}` and replace with `class:external="..."` or a proper variable. The compiler will tell you exactly which files need updating.

---

## New: `class:external` Semantics Clarified

`class:external` is for third-party **component** classes (CMS widgets, payment forms, embedded content). It is NOT for utility CSS frameworks (Tailwind, Bootstrap) — Azumi's `<style>` blocks describe what elements ARE, not what they look like.

**Example messages and tests have been updated** to use component-oriented names (`payment-widget`, `cms-card`) instead of Tailwind examples (`bg-blue-500 px-4`).

## New Features

### Two-Way Form Binding (`bind:value`)
```rust
html! {
    <input type="text" bind:value={state.name} />
    <input type="checkbox" bind:checked={state.agree} />
    <select bind:value={state.choice}>...</select>
    <textarea bind:value={state.bio}></textarea>
}
```
200ms debounce by default. No manual `on:input` handlers needed.

### Keyed List Updates (`@keyed`)
```rust
html! {
    @for item in &items @keyed(item.id) {
        <div class={row_class}>{&item.name}</div>
    }
}
```
Items tracked by identity. Smooth reordering, scroll/focus preserved.

### Transitions (`az-transition:`)
```rust
html! {
    <div az-transition:fade={"true"}>"Fade in/out"</div>
    <div az-transition:slide={"true"}>"Slide open/closed"</div>
    <div az-transition:scale={"true"}>"Scale 0.95↔1.0"</div>
}
```
Custom duration: `az-transition:fade={"duration=500"}` (default: 200ms).

### Loading & Error States
- `az-loading` class auto-added during `az-action` fetch
- `az-error` class auto-added on fetch failure
- Style with CSS: `button[az-loading] { opacity: 0.6; }`

### Debounce on Events
```rust
html! {
    <input az-on:input="debounce=300 call search -> #results" />
}
```

### Lifecycle Hooks
- `az-on:mount` — fires when element appears in DOM
- `az-on:unmount` — fires before element leaves DOM
- `az-on:update` — fires after element content changes

### Extended Form Validation
3 new rules: `max:N`, `min:N`, `pattern:regex`. Now 8 rules total.

### Predicate DSL Simplified
Ternary (`?:`), compound (`&&`, `||`), and numeric comparisons (`<`, `>`, `<=`, `>=`) removed from `az-bind:class` expressions. Use `!`, `==`, `!=`, or truthy checks only. Complex logic belongs in Rust, not HTML attributes.

## Runtime Size
44KB uncompressed / 11KB gzipped (up from 42KB/10KB in v47 with +12 new features).

---

## Upgrade Steps

1. Update Cargo.toml: `azumi-live-ssr-framework = "48.0.0"`
2. Run `cargo build` — the compiler will flag any `class={"..."}` patterns
3. Replace `class={"xxx"}` with `class:external="xxx"` or `class={variable}`
4. Run `cargo test` to verify
